# 决策门槛

需求已明确、影响范围可验证且不改变对外契约时可以直接执行；否则说明影响并等待确认。

## 可直接执行

- 单一页面或私有模块内的文案、布局、样式和无歧义缺陷修复。
- 使用现有 NativeWind 工具类，或在单一模块内迁移静态样式。
- 不改变行为、导出或数据结构的局部重构。
- 运行只读检查、格式检查、类型检查、Lint 和构建验证。
- 修正与当前配置明显不一致的文档事实。

## 必须先确认

### 业务与接口

- 登录流程、权限、角色、路由可达性、重定向、菜单和深链接。
- API 地址、请求响应字段、鉴权、错误语义、重试、缓存和持久化。
- mock 数据改为真实数据，或改变既有默认值、数据模型和用户操作结果。

### 公共结构与配置

- 新增或替换导航、状态管理、请求、图表、缓存等基础设施。
- 修改 `app.json`、应用权限、bundle identifier、图标、启动图或平台行为。
- 升级 Expo、React Native、NativeWind、Tailwind 或其他关键依赖。
- 修改全局 CSS、Tailwind preset、全局设计令牌或公共组件职责。

### 范围与交付

- 大规模删除标准 Expo 模板遗留文件、批量重命名或目录迁移。
- Git 提交、推送、分支、合并、发布及其他外部状态变更。
- 存在两种以上会显著改变产品体验或维护边界的合理方案。

## 确认请求

说明变更场景、受影响用户或文件、主要风险、推荐方案、替代方案及推荐理由。
