# 验证与审查

## 最低验证

- 修改 TypeScript 或 TSX 后运行 `pnpm typecheck`。
- 提交前运行 `pnpm validate`；若格式检查失败，执行 `pnpm format` 后重新运行完整验证。
- 修改界面后，至少验证一个涉及的平台；跨端差异需分别检查相关平台。
- 修改 NativeWind、Metro、Babel、路由或 Web 兼容代码后，运行 `pnpm exec expo export --platform web`。
- 修改提交门禁后，分别验证 lint-staged 配置和一条合法、非法的 Commitlint 消息。
- 当前没有自动化测试；未执行的检查必须如实说明。

## 审查重点

- TypeScript 严格性、空值边界、计时器清理和错误处理。
- 页面是否覆盖加载、空数据、筛选无结果、禁用和重复触发状态。
- 移动端触控区域、键盘行为、滚动、可访问性和响应式布局。
- 是否意外改变路由、应用配置、公共组件、API 契约或全局样式。
- NativeWind 类名是否可被静态扫描，动态样式是否限制在必要范围。
- 新增依赖、配置和脚本是否同步写入 `package.json`、锁文件及协作文档。

## 规则维护

`.agents/` 只描述当前可验证事实与通用约定。产品背景和阶段性方案应写入 `docs/`，不在规则文件中记录决策历史。
