# 项目来源与迁移边界

## PC 源项目

当前移动端项目的业务功能来源于以下 PC 数据中台项目：

```text
/Users/opera/Documents/my-repositories/data-middle-station
```

后续新增、补齐或核对移动端功能时，应优先参考该 PC 项目的业务流程、领域概念、权限模型、接口契约和用户可见行为。

## 迁移原则

- 迁移业务能力和产品语义，不直接复制仅适用于 PC 的页面实现。
- 移动端继续使用 Expo、React Native 和 Expo Router，不引入 PC 项目的 Vue、Element Plus 或浏览器专属 UI 代码。
- 字段名称、权限含义、接口请求和响应契约应以 PC 项目及其服务端实现为准，不在移动端擅自改名或简化。
- 复杂编辑器、拖拽画布、大表格和其他桌面生产力功能，需要根据移动端场景单独设计，不能机械迁移。
- 本地绝对路径用于开发环境中的项目关联；对外协作时应同时提供对应仓库地址或可访问文档。
